SQL Workspace

Write, run, explore.
Like a real IDE.

Multi-tab SQL editor with schema-aware autocomplete, syntax highlighting, inline formatting, and virtual-scrolling results grid. Click any row to inspect details, follow FK references, view mutation history.

  • Schema-aware SQL autocomplete
  • Multi-tab query editor (Cmd+N / Cmd+W)
  • Inline row editing with PK/FK badges
  • FK navigation — click to follow references
  • Visual EXPLAIN plan with index suggestions
  • Query builder for visual query construction
  • Saved queries and query history
  • Data export (CSV, JSON)

Schema Diff Engine

Design → Diff → Apply.
A deployment pipeline
for your schema.

Every change you make in the visual designer generates DDL automatically. The diff engine compares your model against the live database and classifies every operation as safe, unsafe, or breaking.

Apply changes selectively — or batch-apply all safe operations with one click. Versioned FKs and entity definitions are fully supported.

Schema Diff — 4 operations
SAFE CREATE TABLE audit_log (...) ✓
SAFE ALTER TABLE users ADD COLUMN role TEXT ✓
UNSAFE ALTER TABLE orders MODIFY amount INT —
BREAK DROP TABLE temp_cache —
Apply 2 safe Apply all

Get started with Studio

1

Start the ASQL engine

$ go run ./cmd/asqld -addr :5433 -data-dir .asql
2

Launch Studio

$ go run ./asqlstudio -pgwire-endpoint 127.0.0.1:5433 -data-dir .asql
3

Follow the Start Here checklist

Studio opens with a guided onboarding: connect to engine, pick a domain, build a schema, run your first query. Every feature is reachable from there.

4

Use the operator workflows

Inspect temporal history, replica lag, read-routing decisions, failover posture, recovery state, and visual EXPLAIN output from the same desktop workflow.
